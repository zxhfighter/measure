nb-tooltip

type: Directive

selector: [nbTooltip]

exportAs: nbTooltip

Usage

import {TooltipModule} from 'xdesign';

Examples

Tooltip Basic

Tooltip的使用方式有两种，预渲染和动态渲染。

预渲染【推荐】：预渲染指页面初始化后提示框的DOM结构已经存在。

适用场景：

- 大多数场景都可以使用预渲染

- 多个触发元素共享一个提示框时，必须使用预渲染


动态渲染：动态渲染指页面初始化后提示框的DOM结构并不存在，当触发显示时才动态地创建组件。

适用场景：

- 提示内容为简单纯文本。在触发元素的属性上设置文本内容即可。

- 提示内容比较复杂或者被使用的概率较小，需要优化为延迟渲染的场景。提示内容使用ng-template标记包装


当前示例为动态渲染

通用功能：

1. 通过 placement 设置提示框的位置，支持的方向有： 'top' | 'bottom' | 'left' | 'right' | 'top-left' | 'top-right' | 'bottom-left' | 'bottom-right' | 'left-top' | 'left-bottom' | 'right-top' | 'right-bottom' 默认为 bottom 底部显示

2. 通过 trigger 设置提示框的触发类型，可选值为 'click' | 'hover' | 'focus' ，默认为 hover 触发

3. 通过 hasArrow设置是否有箭头

4. 通过 nbTooltipClass设置提示框的 className ，因为提示框是挂载到body下面，可以通过设置className区分提示框的展示形式

5. 通过 nbWidth、nbHeight固定提示框的宽高

Tooltip Dynamic

假设此示例的提示内容比较复杂或者被使用的概率较小，需要按需渲染

提示框的属性（比如定位、宽高）都设置在触发元素上

提示框内容支持任意HTML片段、Angular组件、指令

Tooltip Static

提示框的属性（比如定位、宽高）都设置在提示框元素本身

Tooltip Share

预渲染，列表中多条记录的修改使用同一个提示框

Tooltip Embedded

通过 embedded设置提示框是否为嵌入方式，否则悬浮显示

Tooltip InputFocus

支持 <input type="text"> 的 focus 触发方式， blur 时隐藏

Tooltip OthersTrigger

存在一对触发元素和提示框，但是提示框的显示隐藏是由第三个元素控制的

Tooltip Themes

1.通过 nbTooltipTheme设置提示框主题色，支持的四个主题色分别是 'default' | white' | 'pink' | 'yellow'

2.可以自定义其他主题色，如 nbTooltipTheme="purple"，会为提示框添加一个名为 nb-tiplayer-theme-purple 的样式类
